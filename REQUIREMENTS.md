# 自転車用観光案内 WEB アプリ 要件定義

## 1. プロジェクト概要

走行中の自転車利用者に対し、現在地周辺の観光・飲食スポットを題材とした会話テキストを自動生成し、
音声で案内する WEB アプリ。骨伝導スピーカーでの聴取を想定し、走行中はスマホ画面を見ない・操作しない運用を前提とする。

会話キャラは２人登場させ、２人で会話する
１人目は案内役で、周辺スポット情報をもとに案内をする
もう一人は盛り上げ役で、案内会話をもとに話を盛り上げる

## 2. 対象環境

- **対象端末**: Android スマートフォン
- **対象ブラウザ**: Chrome (Android)
- **接続前提**: 起動時はオンライン。走行中に圏外になる可能性あり
- **配信形式**: PWA（HTTPS 必須）
- **想定機器**: 骨伝導スピーカー、ハンドルマウント

## 3. 技術スタック

| 領域 | 採用技術 |
|---|---|
| フレームワーク | Next.js (App Router) |
| RDS | MySQL ※アカウント管理および会話履歴記録用 |
| 地図 | Google Maps JavaScript API |
| 周辺スポット取得 | Google Places API (New) - Nearby Search |
| 会話テキスト生成 | Gemini API (`gemini-3-flash-preview` 想定) |
| 音声合成 (TTS) | VOICEVOX（さくらの AI Engine 経由）/ (ElevenLabs TTSは初期実装からは外す） |
| オフライン対応 | Service Worker + Cache API |
| 画面常時点灯 | Screen Wake Lock API |
| 位置情報 | `navigator.geolocation.watchPosition` |

## 4. 画面仕様

スマホ縦長レイアウト固定：

```
┌──────────────-────┐
│                   │
│      地図         │ ← 画面上半分。現在位置を中央に追従
│  （現在地中央）   │
│                   │
│                   │
│                   │
│                   │
│                   │
├───────────────-───┤
│セリフ欄1   |キャラ│
│            |画像1 │ ← 画面下半分
│            |      │
|──────────────-────|
│キャラ|セリフ欄2   │
│画像2 |            │
│      |            │
|──────────────-────|
{[一時停止]         }
└──────────────-────┘
```

- 起動時に「開始」ボタンを 1 回タップ
  - このタップで以下を初期化：
    - AudioContext の unlock（モバイル自動再生制限の解除）
    - Screen Wake Lock の取得
    - GPS 取得開始
- 走行中はユーザー操作不要

## 5. 機能要件

### 5.1 地図表示
- スクロール可能な地図を画面上半分に表示
- GPS で取得した現在位置を地図の中央に追従させる
- オフライン時は地図エリアを非表示または「圏外」表示に切替

### 5.2 現在位置取得
- `watchPosition` で連続取得
- Screen Wake Lock により画面オフによる更新遅延を防ぐ

### 5.3 周辺スポット取得（Google Places API）
- **取得範囲**: 現在位置から半径 2 km
- **再取得トリガー**:
  - 会話生成ループ（下記）の度にチェック
  - 前回取得地点から 500 m 以上移動していたら再取得
  - 500 m 未満ならスキップ
- **フィールドマスク**: Basic ティアに収まる最小フィールドのみ
  - `places.displayName`, `places.location`, `places.types` 等
  - 必要に応じ選定後に Place Details を別途呼ぶ
- **クライアントキャッシュ**: 取得済みスポットは次回再取得まで保持する
- **フィルタリング**: 観光・飲食関連の `types` に絞り込み、既出スポットを除外

### 5.4 会話テキスト生成（Gemini API）
- 「１ターン」の定義: みさき（案内役）→ ひより（盛り上げ役）の往復1組（＝下記ループの 1〜14 を1周）を 1ターンとする
- 周辺スポット情報からランダムに１件取得し、１人目（案内役）がそれを元に会話、それに続けて２人目（盛り上げ役）が応答する、を基本とする
- 会話テキストはGeminiで生成する。スポット情報、会話履歴、キャラ毎の設定プロンプトより、Geminiが次の会話を生成する
- キャラ毎の設定テキスト（性格・口調・話し方ルール等のプロンプトテンプレート）を Markdown ファイルで事前定義する
  - 保存場所: `prompts/characters/misaki.md`, `prompts/characters/hiyori.md`
- 会話履歴の保持範囲: 直近 N 時間以内かつ直近 M 件（**デフォルト: 1時間以内かつ直近10件**）。この範囲を Gemini に渡す
- API キーは Next.js Route Handler 経由でサーバ側保護
- 生成した会話テキストはDBに追記する（後日分析するために保持しておく）

なお、会話生成ループは次の繰り返しの想定

1. GPS位置情報取得
2. 初回もしくは前回の位置情報から500m以上移動していれば、Maps API(new)で、周辺スポット情報取得
3. ターンの先頭で、ターゲットスポット情報を更新する（周辺スポット情報からランダムに１件選ぶ）
4. １人目（みさき）の会話生成
　・ターゲットスポット情報
　・プロンプト（キャラ設定）
　・会話履歴
5. 会話テキストをTTSで音声ファイルに変換
6. 会話テキストを再生開始すると共に、音声ファイルの再生を開始する（開始タイミングだけあわせる）
　※テキストは１文字ずつ、タイプライター風に表示する。表示速度は7文字/s（設定調整の可能性あり）
　※テキストと音声のスピードは同期しないが許容
7. テキストと音声再生完了を待つ
8. 10秒ウエイト
9. ２人目（ひより）の会話生成
　・ターゲットスポット情報
　・プロンプト（キャラ設定）
　・会話履歴
10. 会話テキストをTTSで音声ファイルに変換
11. 会話テキストを再生開始すると共に、音声ファイルの再生を開始する（開始タイミングだけあわせる）
　※テキストは１文字ずつ、タイプライター風に表示する。表示速度は7文字/s（設定調整の可能性あり）
　※テキストと音声のスピードは同期しないが許容
12. テキストと音声再生完了を待つ
13. 10秒ウエイト
14. 1に戻る（次のターンへ）


### 5.5 音声合成・再生（VOICEVOX）
- 生成された会話テキストを VOICEVOX で音声化し再生
- 音声のバリエーション（キャラクター）を活用
- 「次のスポット候補を先読みして TTS を事前生成」するパイプラインで体感レイテンシを短縮
- 同一スポットの再訪に備えセリフ＋音声のキャッシュ
- 再生間隔の制御は 5.4 の会話生成ループに従う（各発話後の 10 秒ウェイトを採用。本セクション側では独自のクールダウンは設けない）

### 5.6 音声入力
- 双方向対話は本バージョン非対応

### 5.7 オフライン対応
- **起動時のプリキャッシュ**: Service Worker `install` 時に固定音声ファイルを Cache API へ投入
- **オンライン/オフライン検知**: `navigator.onLine` ＋ `online`/`offline` イベント
- **オフライン時の挙動**:
  - 地図エリアを非表示／「圏外」表示
  - Places / Gemini / VOICEVOX 呼び出しはスキップ
  - キャッシュ済みの固定音声を一定間隔で再生
- **オンライン復帰時**: 通常モードへ自動復帰

### 5.8 ログイン機能

- 最初は自分用に作るが、インターネット上に公開するので、最初の画面のログインフォームを置く
- 初期アカウントは、DB直接登録しておく。今のところ、会員登録やパスワード忘れ対応機能は実装しない

## 6. 非機能要件

### 6.1 ユーザー体験
- 走行時は完全ハンズフリー。設定変更や一時停止は休憩時に操作する
- 信号待ち・休憩時にスポット再検索をしない。休憩時に一時停止可能
- 圏外でも沈黙させず、「ここは圏外のようです」のような固定音声を再生で体験を継続

### 6.2 セキュリティ
- 全 API キー（Google Maps Platform、Gemini、VOICEVOX）はサーバ側で保持
- ブラウザからの直接呼び出しは行わず、Next.js Route Handler 経由

### 6.3 コスト想定（2 時間ライド／自転車 15 km/h）
- Places API: 30〜60 回呼び出し、Basic フィールドマスクで **約 $0.15〜$0.30/ライド**
- Gemini API (`3-flash-preview`): 数円/ライド程度
- VOICEVOX (さくらの AI Engine): 定額想定, (※将来的にElevenLabs TTS APIは有料プラン加入を想定)
- Google Maps Platform の月 $200 無料クレジット内に通常運用は収まる見込み

## 7. 既知の制約・注意点

- **Google Places API の表示制約**: Places 由来のデータは原則 Google Maps と一緒に表示する利用規約に従う
- **地図タイルのオフラインキャッシュ**: Google Maps の利用規約上、タイルの永続キャッシュは制限される。本要件ではオフライン時は地図非表示で回避
- **Android Chrome の自動再生制限**: 最初に必ずユーザータップで AudioContext を unlock する必要がある
- **画面オフ時の `watchPosition`**: スロットルされるため Screen Wake Lock 必須
- **完全バックグラウンド動作は不可**: PWA の範囲では限界があり、必要になった場合は TWA + ネイティブ補助の検討が必要

## 8. 本バージョンのスコープ外

- iOS / Safari 対応
- 音声入力（双方向会話）
- 走行ルートの記録・履歴
- ユーザーアカウント／設定の同期
- 地図タイルのオフライン保持
